02 — Problem

821 pages. One PDF.
No search that understands Apex.

821 pages / ~680K tokens
  • The Apex Developer Guide is the canonical reference — shipped as a single monolithic PDF.
  • Real answers cross-reference each other ("see X on page Y") across sections written independently.
  • Ctrl+F finds keywords. It doesn't find explanations, and it doesn't know when it's wrong.
Not a general chatbot. A grounded specialist that refuses to guess.
?
03 — Approach

Hybrid retrieval + two-layer guardrails, grounded entirely in the source PDF

pdfplumber voyage-3 FAISS BM25 claude-sonnet-5 LangChain

Single-file terminal app — no web UI, no database, by design.

05 — Guardrails

Two layers stand between a question and an answer

L1
Pre-LLM · retrieval confidence. absolute_relevance ≥ 0.30, or refuse — the LLM is never called.
L2
In-prompt · Claude's own backstop. Answer only from the excerpts, cite pages inline as (p. 42), and say so if they don't actually cover the question.
off-topic: −0.11 to 0.23
on-topic: 0.39 to 0.48
06 — Tradeoffs

What I chose, and what I rejected

✗ LangChain EnsembleRetriever (opaque scoring)
✓ Chosen — FusionHand-rolled weighted RRF (0.4 / 0.6, k=60) — keeps per-candidate scores available for the guardrail and section boost.
✗ Router confidence as a hard first gate
✓ Chosen — Router's roleSoft 0.15 re-ranking boost only — "what's the weather like today?" scored higher than the real question "how do governor limits interact with test methods?"
✗ Fused/normalized retrieval score (top hit always ≈1.0)
✓ Chosen — Guardrail signalSeparate raw, per-query-comparable FAISS score of the single closest chunk in the whole corpus.
✗ Dynamic heading-detection parsing
✓ Chosen — Section boundariesHardcoded 6 TOC sections — valid as long as this stays pinned to the Summer '26 edition of the guide.

10 — Impact

Honest status, not inflated metrics

✓ Verified Solo MVP — single commit, no tests, no CI, no users yet.
✓ Verified Corpus (~680K tokens) uses ~0.34% of Voyage's 200M-token free voyage-3 allotment → embedding cost effectively $0.
✓ Verified, caveated Guardrail threshold 0.30 sits in a ~0.16-point gap between on-topic (0.39–0.48) and off-topic (−0.11 to 0.23) — two hand-tested categories, not a benchmark suite.
◇ Qualitative, no number claimed Collapses a slow, cross-referenced 821-page manual search into one question with page citations to verify — no time-saved figure is claimed, since none was measured.
